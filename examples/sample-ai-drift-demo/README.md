# sample-ai-drift-demo

A layered Python service that demonstrates the **AI-Drift Firewall** in
action. Run `./demo.sh` and watch three separate AI-generated "drift PRs"
each get blocked by a different axiom-backed rule, plus one override
scenario for the architect's urgent-bypass case.

Runs in under 5 seconds. Clean output, designed for a 90-second live demo
to the FENG grant panel.

## Architecture

```
src/
├── domain/             # pure values + ports, no I/O
│   ├── order.py
│   └── ports.py        # OrderRepository, Notifier protocols
├── application/        # orchestrates domain + ports
│   └── order_service.py
├── infrastructure/     # concrete adapters for ports
│   ├── db.py           # implements OrderRepository (Postgres)
│   ├── email.py        # implements Notifier (SMTP)
│   └── payments_core.py   # protected — only payments_api may call
└── payments_api/
    └── gateway.py      # the ONLY authorized caller of payments_core
```

Dependency graph is a clean DAG. Layers flow one way: application → domain,
infrastructure → domain. Nothing loops back.

## The three rules

Configured in [qse-gate.toml](qse-gate.toml):

| Rule | Axiom | Catches |
|------|-------|---------|
| `CYCLE_NEW`        | acyclicity (MDL: cycle inflates graph description length)                             | New strongly-connected components in the import graph |
| `LAYER_VIOLATION`  | layering (MDL: high-level layer compressed independently of low-level)                 | Forbidden edges, e.g. `domain → infrastructure`       |
| `BOUNDARY_LEAK`    | encapsulation (MDL: protected partition compressed through its API surface only)       | Edges into a protected module from an un-whitelisted caller |

Each rule cites its axiom in the PR comment. Overrides are logged to
`artifacts/gate-telemetry.jsonl` for audit.

## Running the demo

```bash
pip install -e ../..        # install qse-pkg from the repo root
./demo.sh
```

You'll see five sections:

1. **Clean baseline** — the untouched service passes all three rules.
2. **cycle_new drift** — domain.order picks up a reverse-import to
   application.order_service. Ring closes. Gate blocks. Note: the same
   overlay also trips `LAYER_VIOLATION` (domain→application is forbidden)
   because adding a cycle across layer boundaries tends to break both rules
   at once. Honest rules, not theater.
3. **layer_violation drift** — domain.order reaches directly into
   infrastructure.db. Same double-hit behavior.
4. **boundary_leak drift** — a new analytics module bypasses `payments_api`
   to call `payments_core` directly. Clean single-rule violation.
5. **Override showcase** — architect sets `[skip-qse]` in the commit
   message. Gate exits 0 with violations still logged to telemetry.

## Using this in your own CI

Add to `.github/workflows/ci.yml`:

```yaml
name: CI
on: [pull_request]
jobs:
  drift-gate:
    uses: PiotrGry/qse-pkg/.github/workflows/ai-drift-gate.yml@main
    with:
      config:   .github/qse-gate.toml
      path:     .
      base-ref: ${{ github.base_ref }}      # Δ mode compares against PR target
```

The reusable workflow:

- Runs `qse-gate` with `--base-ref` so `CYCLE_NEW` only flags **new** rings
  (not pre-existing ones, even in Event-Bus / CQRS-saga style architectures).
- Posts a sticky PR comment with every violation (rule, edge, axiom, fix hint).
- Uploads the JSON report, Markdown comment, and JSONL telemetry as
  workflow artifacts.
